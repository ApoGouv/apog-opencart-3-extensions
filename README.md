# Apog OpenCart 3 Extensions Development Toolkit

![PHP](https://img.shields.io/badge/PHP-8.1+-blue)
![License](https://img.shields.io/badge/license-Apache%202.0-green)

A modular **framework and CLI toolkit** for building structured, scalable OpenCart 3 extensions.

It provides a structured development environment for building, generating, and packaging:
- Payment
- Shipping
- Total extensions

Powered by a shared core library (`apog_core`).

---

## 🚀 Quick Start

```bash
# 1. Generate a payment module
php bin/generator.php --type=payment --code=cod --name="Cash on Delivery"

# 2. Build it
php bin/build.php --type=payment --code=cod

# 3. Install the generated package from /dist in OpenCart
```

---

## ✨ Features

* CLI-based extension generator
* CLI-based build/packaging system
* Unified architecture for:
  * **Payment**
  * **Shipping**
  * **Total** extensions
* Shared **core library** (`apog_core`) used across all extensions
* Built-in support for:
  * Geo Zones
  * Store / Customer Group restrictions
  * Shipping & Payment exclusions
  * Payment-bound totals (fees)
* Multi-language ready (GR / EN)

---

## 📦 Project Structure

- `bin/` - CLI tools (generator & build system)
- `src/` - Source modules (sample extensions build with this framework + CLI toolkit)
- `dist/` - Generated `.ocmod.zip` packages
- `generators/` - Template system for extensions scaffolding
- `core/` - Core library module (shared system logic)

## ⚙️ CLI Tools

### 1. Generator

Creates a new module scaffold compatible with OpenCart 3 installer format
(`upload/` structure + `install.xml`).

It supports the `core` library that generates the `apog_core` files.
> ⚠️ All extensions generated by this toolkit depend on the shared `apog_core` library.

And can generate extensions for `payment`, `shipping` and `total`.

#### Usage
```bash
  php bin/generator.php --type=core [--force]
  php bin/generator.php --type=shipping --code=<code> --name="Name" [--force]
  php bin/generator.php --type=payment  --code=<code> --name="Name" [--force]
  php bin/generator.php --type=total    --code=<code> --name="Name" [--binding_payment=<code>] [--force]
```

#### Parameters and Options

| Parameter | Description                                                        |
| --------- | ------------------------------------------------------------------ |
| `--type`  | Module type: `core`, `payment`, `shipping`, `total`                |
| `--code`  | Module code (required except core, must be at least 3 characters long) |
| `--name`  | Module display name (required except core)                        |
| `--binding_payment` | Payment module code (e.g. cod). Only for `total` modules |
| `--force` | Overwrite existing module in `src/` folder                         |
| `--help`  | Shows help message                                                 |


#### Examples

##### Payment (COD)

```bash
php bin/generator.php --type=payment --code=cod --name="Cash on Delivery"
```

##### Payment (Pay at Store)

```bash
php bin/generator.php --type=payment --code=pay_at_store --name="Pay at store"
```

##### Shipping (Store Pickup)

```bash
php bin/generator.php --type=shipping --code=pickup_at_store --name="Pickup at store"
```

##### Total (COD Fee with binding)

```bash
  php bin/generator.php --type=total --code=cod_fee --name="COD Fee" --binding_payment=cod
```

> ⚠️ Binding is recommended via CLI. Manual setup requires editing admin controller.

---

#### 📂 Module Naming Convention

> All generated extensions use the `apog_` prefix in the following format:

```
apog_<type>_<code>
```

Examples:

* `apog_payment_cod`
* `apog_shipping_acs`
* `apog_total_cod_fee`

Core:

* `apog_core`

### 2. Build Tool

Builds one or multiple modules into installable OpenCart `.ocmod.zip` packages.

#### Usage
```bash
  php bin/build.php --type=core
  php bin/build.php --type=shipping --code=<code>
  php bin/build.php --type=payment  --code=<code>
  php bin/build.php --type=total    --code=<code>
  php bin/build.php --all
```

#### Parameters and Options

| Parameter | Description                                                        |
| --------- | ------------------------------------------------------------------ |
| `--type`  | Module type: `core`, `payment`, `shipping`, `total` *              |
| `--code`  | Module code (required except core, must be at least 3 characters long) * |
| `--all`   | Build all modules found in `src/` folder                           |
| `--help`  | Shows help message                                                 |

*For `--all` mode, both `--type` and `--code` are not required.

#### Output

Packages are generated in:

```
dist/<module_name>.ocmod.zip
```

#### Examples

##### Package core library

```bash
  php bin/build.php --type=core
```

##### Package payment extension with code `cod`
```bash
  php bin/build.php --type=payment --code=cod
```

##### Package all extensions (including code library if available) in the `src/` dir
```bash
  php bin/build.php --all
```

## 🏗 Core Library

The `apog_core` module provides shared infrastructure used across all extensions:

- Payment services & validation
- Shipping calculators & services
- Order total calculators
- Shared utilities & traits
- Autoloading & bootstrap logic

> ⚠️ Must be installed before any other module.

## 🚀 Development Workflow

1. Generate module:

```bash
php bin/generator.php --type=payment --code=cod --name="COD"
```

2. Implement logic in:

```
src/apog_payment_cod/
```

3. Build package:

```bash
php bin/build.php --type=payment --code=cod
```

`--all builds everything in src/`

4. Install `.ocmod.zip` into OpenCart


## 📥 Installation (OpenCart Admin)

1. Go to:

   ```
   Extensions → Installer
   ```

2. Upload:

   ```
   apog_cod_fee.ocmod.zip
   ```

3. Then go to:

   ```
   Extensions → Modifications
   ```

4. Click:

   ```
   Refresh
   ```

5. Finally:

   ```
   Extensions → Extensions
   Select type (Payment / Shipping / Total)
   Install & Enable your extension
   ```
